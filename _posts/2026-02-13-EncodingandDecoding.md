## 인코딩 디코딩 
- **인코딩** : 데이터를 특정한 형식으로 변환하는 것을 말한다.
    - 데이터의 크기를 줄이거나 컴퓨터가 이해하기 쉽게 변환할때 사용한다.
    - 인코딩은 암호화와 유사하지만 암호화는 데이터의 기밀성을 위한 목적이며 비밀키가 있어야 원문으로복구가 가능하지만 인코딩은 누구나 표준화된 방식을 사용하여 디코딩을 하여 원문을 구할 수 있다.

- **디코딩**: 인코딩된 데이터를 원래의 값으로 되돌리는 것을 말한다.

### 아스키 코드 (ASCII)
- 컴퓨터는 0과 1만을 이용하여 정보를 처리하고 표현한다.
- 우리가 입력하는 모든 글자가 0과 1로 변환되려면 숫자로 표현되어야 한다. 이를 위한것이 아스키 코드이다.
- ASCII는 정보교환을 위한 미국표준 코드로 문자를 숫자로 변환하는 문자 인코딩의 표준이다. 
- 이를 이용하여 인코딩 하면 서로 다른 장치간 데이터 전송을 더 쉽게 할수 있다.
- 아스키 문자 1개는 1바이트 크기로 7비트로 문자를 표현하고 1비트는 오류를 체크한다. 
-  **$2^7$**=128 가지의 문자표현이 가능하다. 
- 각 문자는 0~127 까지의 10진수 값을 가진다.

### 유니코드 
- 유니코드는 영어 뿐만 아니라 전세계 모든 언어의 문자에 고유한 번호를 부여하는 국제 표준코드이다.
- 아스키 코드보다 용량을 크게 확장하여 최대 32비트로 문자 1개를 표현한다. 
- 유니코드 안에 아스키 코드를 포함한다. 

### URL 인코딩 
- 웹에서 사용되는 URL은 특정 형식의 문자열만 허용한다.
- 알파벳 대소문자,숫자,일부 특수문자만을 포함 할수 있다.
- 웹 브라우저로 부터 받은 URL문자열을 유요한 형식으로 변한 하는 것은 URL인코딩이다.
- URL인코딩을 통해 문자열을 인터넷 전송 가능한 형식으로 변환한다. 
- URL인코딩은 % 기호 뒤에 해당 문자의 아스키 코드 16진수 값을 붙여 나타낸다. 

### BASE64 인코딩
- 이진 데이터를 아스키 문자로 구성된 텍스트로 변화하는 인코딩 방식이다.
- 총 64개의 아스키 문자가 인코딩에 사용된다.
- 64개의 아스키 문자는 알파벳 대소문자(52자) , 숫자(10자),+,/ 이다.
#### Base64 인코딩 방식 
1. 원본 이진 데이터를 비트 나열로 표현하고 이를 6비트씩 끊어서 묶는다.
    - 만약 6의 뱌수가 아닐 경우 0을 뒤에 추가하여 6의 배수로 만든다.
2. 6비트 묶음을 수로 변환한뒤 base64 테이블에서 해당 문자를 찾아 치환한다.
3. 치환 후 글자 수가 4의 배수가 되도록 문자 `=`를 반목하여 뒤에 추가한다. (이는 **Padding** 이라고 한다.)
