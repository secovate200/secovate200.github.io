## Linux
- 리눅스는 리누스 토발즈에 의해 개발된 운영체제 이다. 
- 리눅스는 오픈소스이며 현재는 리누스 토발즈 뿐만 아니라 다수의 기여자들이 리눅스의 발전에 기여하고 있다.

## Shell 
- 운영체제는 **User Level** 과 **Kernel Level** 로 구분이 된다.
    - **User Level** : 사용자가 조작할 수 있는 부분 
    - **Kernel Level** : 일반 사용자가 조작할수 없는 부분 
- 사용자가 운영체제를 온전히 사용하기 위해서는 커널과 유저 사이의 연결해주는 프로그램이 필요하며 이를 **Shell (셸)** 이라고 부른다. 
### Shell 의미
- 컴퓨터 공학에서의 Shell은 운영체제와 상호작용할수 있는 인터페이스를 제공하는 소프트웨어라는 의미이다.
- Shell은 GUI Shell와 CLI Shell로 나눌수 있다.
    - GUI Shell
        - 사용자들이 직관적으로 기능을 파악하고 사용하는 것에 장점이 있지만 CPU나 GPU를 많이 소모한다는 단점이 있다. 
    - CLI Shell 
        - 상대적으로 직관적이지 않으며 주요 명령어를 알고 있어야 한다는 단점이 있으나 정밀한 작업이나 빠른 실행 속도가 장점이다. 

### Shell 의 종류
1. Terminal 
    - CLI Shell을 GUI Shell 위에서 사용할수 있도록 한 프로그램이다.
    - 기본적으로 Bash를 사용한다. 
        - Bash 이외에도 다양한 Shell이 존재한다. 

## Sh
- Sh는 기본적으로 아래와 같은 기능을 지원한다.
    1. 기본적인 프로그램 실행
    2. 기본적인 입출력
    3. 환경변수 설정
    4. Shell 프로그래밍
## Bash 
- Sh를 대체하기 위해 등장하였다. 
- Bash는 sh의 기능에 더하여 아래와 같은 내용을 추가로 지원한다.
    - 이전에 실행한 명령을 저장해둠(명령 히스토리 지원)
    - tab 자동완성 지원
    - bash로그인 시 아래 시작파일이 자동으로 실행된다.
        1. `/etc/profile`이 존재하면 읽은 후 실행
        2. `~/.bash_profile`,`~/.bash_login`, `~/.profile`을 순차적으로 탐색 하여 존재하면 읽은 후 실행
    - bash종료시 아래 파일이 자동으로 실행된다.
        1. `~/.bash_logout`이 존재하면 읽은 후 실행
    - Shell에서 bash실행시 아래 시작파일을 자동으로 실행
        1. `/etc/bash.bashrc`와 `~/.bashrc`가 존재할 경우 실행
## ash & dash
- bash보다 경량화 된 Unix Shell이다.
- bash보다 가볍고 라이브러리 의존성이 상대적으로 적다. 
- root사용자의 shell, `/bin/sh`대체 와 같은 상황에 주로 사용된다.
